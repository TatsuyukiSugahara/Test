# ゲームアプリケーションコア 設計ドキュメント

> 対象コミット: `144ae8a` / 最終更新: 2026-07-03

対象: `aqEngine/`(Engine / aq.h)、`aqEngine/Core/`、`aqEngine/Platform/`、`Game/Application/`
関連: [Xbox移植設計](../aqEngine/Platform/Xbox移植設計.md)、[レンダリング設計](01_レンダリング設計.md)、[ECS設計](04_ECS設計.md)、[マルチスレッド設計](05_マルチスレッド設計.md)

エンジンの**起動・メインループ・ライフサイクル・プラットフォーム抽象**と、その上に乗る**ゲーム側の実装骨格**(アプリ/シーン/アクター)を扱う。ここが「DX11/DX12/Xbox/UWP ハイブリッド」を配線する結節点。

---

## 0. 全体構造(起動から終了まで)

```
[エントリ]  Main.cpp(Win32 WinMain)  /  UWPMain.cpp(IFrameworkView・道A)
      │  IPlatform 実装を生成(PlatformWin32 / PlatformUWP)
      ▼
Engine（シングルトン）
  Create() → CreateApplication<app::Application>() → Initialize(param) → RunGame() → Finalize()
      │
      ├─ Initialize: Memory → Window(Platform) → GraphicsAPI(GraphicsDevice::Create<Impl>)
      │              → ThreadPool → Sound → AudioDirector → application_->Initialize/Register
      │
      └─ RunGame: while(platform_->PumpEvents()) { Update(); }
            Update(): gameTimer_.Tick() → [ToggleMainRT] → application_->Update()
                      → Sound/Audio Update → application_->FlushRender() → ResetStackAllocator
      ▼
Application（aq::Application : IApplication）… エンジンサブシステムの初期化/更新/終了の基底
      │  OnInitialize / OnUpdate / OnRegister / OnPreRender を override
      ▼
app::Application（ゲーム実装）→ SceneManager → IScene（BattleScene 等）
```

**ハイブリッドの 3 つの分岐点**:
1. **エントリ/ウィンドウ/ループ** … `IPlatform`(Win32 / UWP)。
2. **グラフィックス API** … `GraphicsDevice::Create<TImpl>()`(D3D11 / D3D12 / Vulkan)を `ENGINE_GRAPHICS_*` で選択。
3. **リソース予算** … `PlatformBudget.h`(Win32 = 無制限 / Xbox UWP = 5GB・6 worker・2GB/ファイル)。

## 1. ブートストラップ(`aqEngine/`)

| ファイル | 責務 |
|---|---|
| `aq.h` | **プリコンパイル済みヘッダ相当の共通インクルード + API 選択の中枢**。`ENGINE_GRAPHICS_*` マクロ検証(ちょうど 1 つ)、API 別の `#include`/`#pragma comment(lib)`(d3d11/d3d12/dxgi/vulkan)、DirectInput、DirectXTex(デスクトップ=prebuilt/MT / UWP=NuGet/MD 分岐)、標準ライブラリ、Math/Util、`StartupLog`(UWP 実機の起動診断ログ)を集約。**ここを見れば「どの API・どのプラットフォームでビルドされるか」が分かる**。 |
| `aq.cpp` | `#include "aq.h"` のみ(PCH 生成用 TU)。 |
| `RenderConfig.h` | レンダリング同期モード `AQ_RENDER_PIPELINED`(直列/非同期)の単一定義点。全 TU から見える。 |
| `Engine.h` / `Engine.cpp` | **エンジン中枢(シングルトン)**。`InitializeParameter`(platform/画面・描画解像度/メモリ設定)。`Initialize` は Memory→Window→GraphicsAPI→ThreadPool→Sound→AudioDirector→Application の順で初期化。`RunGame` は `platform_->PumpEvents()` が false を返すまで `Update()`。`Update()` は `gameTimer_.Tick()` →(非同期時)`ToggleMainRenderTarget()` → `application_->Update()` → Sound/Audio Update → `FlushRender()` → StackAllocator リセット。`GetMainRenderTarget(Handle)`(RT ダブルバッファのトグル管理)、`GetContentRoot`(Platform 委譲)、`GetHWND`(Win32 専用の最後の接合点)。 |

**メインループの要点(レビュー)**:
- **RT トグル**: `AQ_RENDER_PIPELINED` 時、フレーム N/N+1 で別メイン RT を使う(`ToggleMainRenderTarget`)。これがレンダースレッドの 1 フレーム重複の前提。
- **スレッド境界**: `FlushRender()` 後まで描画 API はレンダースレッドに閉じる。メインは `Submit` 以降コンテキストを触らない。
- **初期化順序**: Memory を最初に(ウィンドウ/グラフィックス初期化中の new/delete もエンジンアロケータ管理下)。Bullet アロケータフックは Bullet 型生成前。

## 2. アプリケーション基底(`aqEngine/Core/`)

| ファイル | 責務 |
|---|---|
| `IApplication.h` | **アプリ抽象 IF**。`Initialize(RenderContext&)`/`Finalize`/`Update`/`FlushRender`/`Register`。`Update` は RenderContext を受け取らない(描画は `RenderCommandList` に積んで `RenderThread::Submit`)。 |
| `Application.h` / `Application.cpp` | **エンジンサブシステムを配線する基底クラス**(`aq::Application : IApplication`)。`Renderer`/`RenderThread`/`HiZRenderer` を所有。`Initialize` で ResourceManager/EntityContext/InputManager/UIContext/CameraManager/LightManager/(debug)DebugUI/ComponentRegistry/ImGui を初期化し、機能ゲート(`IsComputeSupported`)で Hi-Z・クラスタカリングを条件付き有効化。`Update` は Input→ECS→Resource→`OnUpdate`→UI→Camera→`Render`。`Render` は ImGui NewFrame(バックエンド別 `#ifdef`)→`OnPreRender`→コマンドリスト構築→`BuildRenderFrame`→`BuildCommandList`→`Submit`。`Register` はコア System(HierarchicalTransform/Animation/Spawn/Render)登録+依存設定+`FinalizeRegistration`。ゲームは `OnInitialize/OnFinalize/OnUpdate/OnRegister/OnPreRender`(+debug の `OnImGuiRender/OnDebugRender*`)を override。 |
| `DebugUI.h` / `DebugUI.cpp` | **デバッグ UI レジストリ**(`AQ_DEBUG_IMGUI` 限定)。`IDebugRenderable` を集約し `DebugRenderMenuAll/DebugRenderAll` で一括描画。ECS System とは独立したパネル(レンダリング/UI エディタ/プロファイラ/Prefab 等)の登録先。 |
| `IDebugRenderable.h` | デバッグパネルの抽象 IF(`GetDebugLabel`/描画)。 |

**設計意図**: エンジンの共通処理を `aq::Application` に集約し、ゲームは `On*` フックだけを書く(テンプレートメソッドパターン)。UI/Occlusion への依存は `std::function` コールバックで疎結合(`Renderer` 参照)。

## 3. プラットフォーム抽象(`aqEngine/Platform/`)

| ファイル | 責務 |
|---|---|
| `IPlatform.h` | **プラットフォーム抽象 IF**。`WindowDesc`、`CreateMainWindow`(Win32=HWND / UWP=CoreWindow を `void*` で返す)、`PumpEvents`(終了要求で false)、`OnSuspend/OnResume`(UWP PLM ライフサイクル)、`GetContentRoot`(アセット基点。Win32=ソースツリー / UWP=install フォルダ)。 |
| `PlatformWin32.h` / `.cpp` | **Win32 実装**。通常ウィンドウ生成 + メッセージループ(`PeekMessage`)。道A の回帰確認用の主経路。 |
| `PlatformUWP.h` / `.cpp` | **UWP 実装(道A 本命)**。`CoreApplication/CoreWindow` モデル。ウィンドウは UWP ランタイムが先に生成(`IFrameworkView::SetWindow`)するため既存 CoreWindow をラップ。イベントは Dispatcher でポンプ、`Closed` で終了要求。`Suspending/Resuming`(PLM)トークンを保持。`GetContentRoot` はパッケージ install フォルダ。`GetPixelSize` でスワップチェーン初期サイズ。 |
| `PlatformBudget.h` | **プラットフォーム別リソース予算(コンパイル時定数)**。`ResourceBudget`(メモリ上限/StackAllocator サイズ/ThreadPool worker 数/単一ファイル最大)。Win32 = 無制限・論理コア数、Xbox UWP「Game」分類 = 5GB・stack 8MB・worker 6・2GB/ファイル。`Engine::Initialize` が `threadPoolWorkerCount` を、Memory が `stackSizeBytes` を使う。 |
| `Xbox移植設計.md` | Xbox/UWP 移植の設計(道A=UWP / 道B=GDK、フェーズ計画、FL フォールバック)。 |

**ハイブリッドの肝**: `Engine` は `IPlatform*` しか知らない。エントリ(`Main.cpp` / `UWPMain.cpp`)が Win32/UWP の実装を選んで注入する。API 選択(`GraphicsDevice::Create<Impl>`)と直交しているため、「UWP × D3D12」「Win32 × D3D11」等の組み合わせが自由。

## 4. ゲーム側実装(`Game/Application/`)

### 4.1 エントリとアプリ

| ファイル | 責務 |
|---|---|

## 5. ライフサイクル詳細

### 5.1 初期化順序(`Engine::Initialize`)
```
MemoryManager → PhysicsAllocatorHook → Window(Platform) → GraphicsAPI(GraphicsDevice::Create<Impl> + Initialize)
→ ThreadPool(予算 worker 数) → SoundEngine(Create<DefaultBackend>) → AudioDirector
→ application_->Initialize(各エンジンサブシステム + ImGui + 機能ゲート) → application_->Register(System 登録) → gameTimer_.Initialize
```
各段で `StartupLog` を吐き、UWP 実機(デバッガ接続不可)で到達点を LocalState/startup.log から特定できる。

### 5.2 終了順序(`Engine::Finalize`)
```
application_->Finalize(FlushRender で描き切ってから RenderThread 停止) → AudioDirector → SoundEngine
→ GraphicsDevice → ThreadPool → MemoryManager
```
依存の逆順で破棄(AudioDirector は SoundEngine より先 = SoundStream が SoundEngine を参照するため)。

### 5.3 PLM(UWP サスペンド/レジューム)
`PlatformUWP` が Suspending/Resuming を受け、`Engine`/`GraphicsDevice::OnSuspend/OnResume` へ伝播。suspend 時に GPU アイドル化・メモリ解放(バックグラウンド 128MB 制限対応・Phase 4)。

## 6. レビュー時のチェックポイント

- [ ] **エントリの排他**: Win32 と UWP のエントリ(`Main.cpp`/`UWPMain.cpp`)が `AQ_PLATFORM_UWP` で正しく排他になり、二重定義しないか。
- [ ] **抽象の直交性**: `IPlatform`(ウィンドウ/ループ)と `GraphicsDevice`(API)が独立し、任意組み合わせが成立するか。`Engine` に Win32/UWP 固有型が漏れていないか(`GetHWND` は Win32 限定の明示接合点)。
- [ ] **初期化/終了順序**: 依存の逆順破棄、Memory 最初/最後の徹底、FlushRender→RenderThread 停止の順序。
- [ ] **RT トグルとパイプライン**: `AQ_RENDER_PIPELINED` 時の `ToggleMainRenderTarget` がフレーム重複の前提を満たすか。
- [ ] **機能ゲート**: `IsComputeSupported()` による Hi-Z/クラスタ/Bloom の条件付き有効化が、FL11_0 未満(= Xbox One UWP App 分類の FL10_0 経路)で正しく落ちるか。
- [ ] **予算反映**: `PlatformBudget` の worker 数/stack/メモリ上限が実際に使われ、Xbox の制約に沿うか。
- [ ] **On* フックの粒度**: ゲーム側が `aq::Application` の共通処理に手を入れずフックだけで完結できているか(テンプレートメソッドの純度)。
- [ ] **StartupLog**: UWP 実機で初期化到達点を追える診断が主要段に入っているか。
