## 0. 設計の全体像(要約)

グラフィックスと**同じ Bridge パターン**。コア(`aq::sound`)は API 非依存で、`ISoundBackend`/`ISoundVoice` の実装を差し替える。

```
Game / ECS
   │  ResourceManager::Load<SoundClip>() → SoundEngine::Play(clip) / SoundSource
   ▼
SoundEngine (aq::sound, コア・シングルトン・Bridge)
   ├─ SoundListener / SoundSource / SoundClip / SoundStream / SoundHandle
   ├─ Mixer3D        3D パラメータ → per-voice ゲイン/周波数比 をコア側で計算
   └─ SoundBus       Master / BGM / SE / Voice グループ音量
   │
   ▼  impl_ : ISoundBackend  (Bridge — IGraphicsDeviceImpl と同じ立ち位置)
   ├─ XAudio2SoundBackend  (Windows・実装済み)
   │     └─ XAudio2SoundVoice : ISoundVoice
   └─ (将来) OboeSoundBackend (Android)
```

**ハイブリッドの肝**:
- 3D 音響(距離減衰・パン・ドップラー)は **X3DAudio を使わずコア(`Mixer3D`)で計算**し、プラットフォーム間で聞こえ方を統一。バックエンドには「ボイス再生 + per-channel ゲイン + 周波数比」だけを渡す。
- バックエンド選択は `SoundBackend.h` の `#define`(Win=XAudio2 / Android=Oboe)。Windows ビルドは Oboe を一切リンクしない。
- Xbox/UWP でも XAudio2(`xaudio2_9`)がそのまま動く。

## 1. 型・IF・ハンドル(コア基盤)

| ファイル | 責務 |
|---|---|
| `SoundTypes.h` | API 非依存の型集約。`SoundFormat`/`AttenuationModel`(Linear/Inverse/Exponential)/`LoopRegion`/`SoundClock`/`SoundBusId`(Master/BGM/SE/Voice)等。座標は `Math/Vector` を流用。 |
| `SoundFwd.h` | 前方宣言・`RefSoundClip = shared_ptr<const SoundClip>` 等のエイリアス。 |
| `SoundHandle.h` | **世代付き軽量ハンドル**。`SoundHandle`(再生インスタンス)/`SoundSourceHandle`(3D ソース)。破棄後アクセスは世代不一致で安全に no-op。ECS は生ポインタでなくこれを保持。 |
| `ISoundBackend.h` | **バックエンド抽象**(Bridge Implementor)。`Initialize/Finalize`、ボイス工場 `CreateVoice(format, bus)`、`SetBusVolume/SetMasterVolume`、`GetOutputClock`(デバイス出力クロック = A/V 同期の latency anchor)、`Update`(ポンプ)。 |
| `ISoundVoice.h` | **1 音源インスタンス抽象**。`SubmitBuffer`(ストリーミング・内部コピー・背圧 `WouldBlock`)/`SubmitClipRegion`(常駐クリップのゼロコピー・`RefSoundClip` 保持で生存保証)/状態機械(`Start/Pause/Resume/Stop`)/`SetVolume`/`SetFrequencyRatio`(ピッチ・ドップラー)/`SetOutputMatrix`(3D パン結果)/`GetConsumedFrames`(media clock anchor)/`GetQueuedBufferCount`(背圧)。 |
| `SoundBackend.h` | **バックエンド選択の `#define`**。Win=XAudio2 / Android=Oboe を `using DefaultSoundBackend` で切替。 |
| `ISoundVoice.h`/`ISoundBackend.h` のリアルタイム契約 | PCM 所有権・状態機械・ループの submit 時確定・Oboe コールバック寿命・SPSC スレッド集約は [Sound設計.md §3](Sound設計.md) 参照。 |

## 2. コア再生・3D(`aq::sound`)

| ファイル | 責務 |
|---|---|
| `SoundEngine.h/.cpp` | **サウンドの統括窓口(シングルトン・Bridge)**。`Create<TImpl>()` でバックエンド注入。`Play(RefSoundClip, bus)`(2D ワンショット)/`CreateSource/Resolve/DestroySource`(3D 発音体プール・世代管理)/`OpenStream`(BGM/動画ストリーム生成)/`GetListener`/`SetBusVolume/SetMasterVolume`。`Update(dt)` で 3D 更新・ストリーム供給・ボイス回収。 |
| `SoundSource.h/.cpp` | **3D 発音体**。位置/速度/減衰モデル/min-maxDistance/ピッチ、`Play(LoopRegion)/Pause/Resume/Stop`。内部に `ISoundVoice` を 1 本持ち、`Mixer3D` の結果を反映。3D 位置持ちは mono 入力限定。 |
| `SoundListener.h` | **3D リスナー**(= カメラ)。位置/前方/上/速度。 |
| `Mixer3D.h/.cpp` | **コア側 3D 計算**。距離減衰 → パンニング(1×dst 出力行列)→ ドップラー(周波数比)→ バス/マスターゲイン合成。X3DAudio 非依存でプラットフォーム間の一致を担保。 |
| `SoundClip.h/.cpp` | **常駐 PCM リソース**(SE)。デコード済み PCM + `SoundFormat` + 既定 `LoopRegion`。`res::ResourceBase` 派生で非同期ロード/キャッシュに乗る。`SoundClipLoader` が拡張子で decoder を選ぶ。不変・複数ソース共有。 |
| `SoundStream.h/.cpp` | **ストリーミング元**(BGM/動画音声)。decoder + リングバッファ + streaming 状態。`SoundEngine::Update` がリングへ供給(コピー投入)。ループは stream 側の decoder seek で行う。`GetPlaybackClock()`(media clock)= A/V 同期の権威。キャッシュ非共有の実体。 |
| `VolumeEnvelope.h` | 音量エンベロープ(フェードイン/アウト)。 |

## 4. バックエンド(`Sound/XAudio2/`)

| ファイル | 責務 |
|---|---|
| `XAudio2SoundBackend.h/.cpp` | **XAudio2 バックエンド実装**。`XAudio2Create`→`CreateMasteringVoice`→バス分 `CreateSubmixVoice`。`CreateVoice` は `CreateSourceVoice` して指定バスへ送出。`GetOutputClock` は `GetPerformanceData` で latency/underrun を埋める(mastering の SamplesPlayed は device clock として使わない)。 |

## 9. レビュー時のチェックポイント

- [ ] **抽象の純度**: `aq::sound` コアや ECS に XAudio2 型が漏れていないか(`ISoundBackend`/`ISoundVoice` に閉じているか)。
- [ ] **PCM 生存**: `SubmitClipRegion` が `RefSoundClip` を保持し、ResourceManager の unload・Finalize 順に依存せず PCM が生きているか(XAudio2 はデータをコピーしないため重要)。
- [ ] **オーディオスレッド無ブロック**: コールバック内でロック/確保/IO をしていないか。供給はロックフリーリングか。
- [ ] **ハンドルの安全性**: 再生終了ボイス/破棄ソースへのアクセスが世代チェックで no-op になるか。ECS が生ポインタを持っていないか。
- [ ] **3D の一致**: `Mixer3D` の距離減衰/パン/ドップラーがプラットフォーム非依存(コア計算)で完結しているか。
- [ ] **プラットフォーム分離**: `MFDecoder` 等の Windows 専用コードが `#define` で Android ビルドから除外される構造か。
